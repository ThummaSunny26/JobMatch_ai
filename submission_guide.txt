## Technical Deep Dive (Preparation for Review)

### 1. Prompting Techniques
The system prompt in [prompts.py](file:///c:/Users/Admin/Desktop/life_frontend/agent/prompts.py) is designed using the **ReAct (Reasoning + Acting)** framework. It explicitly defines:
- **Identity**: Sets the persona as "JobMatch AI", an autonomous recruiter.
- **Workflow**: Provides a step-by-step logic (Search → Score → Save → Verify) to minimize hallucinations.
- **Constraint-Based Scoring**: Defines specific numeric ranges for edge cases (e.g., 40-55 for missing profiles) to ensure consistent evaluation.

### 2. Tool Guidance
The agent uses **function calling** (tool binding). The LLM decides to call a tool based on the docstrings and argument schemas defined in [agent.py](file:///c:/Users/Admin/Desktop/life_frontend/agent/agent.py). For example, if the LLM's "Thought" determines it needs to find a LinkedIn profile, it identifies the `search_candidate` tool as the matching action.

### 3. Agentic Flow (LangGraph)
The flow is orchestrated in [agent.py](file:///c:/Users/Admin/Desktop/life_frontend/agent/agent.py) using a `StateGraph`:
- **State**: A `TypedDict` that tracks the message history, iteration count, and evaluation data.
- **Nodes**: `agent_node` (reasoning) and `tool_node` (execution).
- **Conditional Edge**: The `router` function checks if the LLM generated `tool_calls`. If yes, it routes to `tools`; if no (or if max iterations reached), it routes to `END`.
- **The Loop**: The cycle between `agent` and `tools` continues until the agent provides a final recommendation or hits the 8-iteration limit.
